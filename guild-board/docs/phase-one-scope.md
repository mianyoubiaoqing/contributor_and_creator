# 第一阶段制作范围

## 目标

为 10 人 Game Jam 团队提供一个可运行的贡献结算工作台。系统先冻结贡献比例，不在结算阶段直接决定奖金金额。

## 已实现流程

1. 项目创建
   - 项目名、活动名、项目阶段。
   - 引擎、引擎版本、目标平台、仓库地址。
   - 依赖列表。
   - Markdown 协作文档导入和编辑。

2. 任务分配
   - 任务按设计、程序、美术、音频、媒体、QA、制片、本地化、文档分类。
   - 告示板状态：开放领取、已领取、进行中、提交验收、返工、已通过。
   - 支持拖拽改变任务状态。
   - 表格中可调整完成度、质量系数、准时系数、证据系数。

3. 成员互评
   - 评分维度：交付可靠性、协作质量、专业贡献、质量意识、项目支持。
   - 支持自评与互评，但自评是否计入后续可以在云端规则中配置。

4. 结算
   - 实时贡献比例。
   - 预结算快照。
   - 冻结贡献比例。
   - 贡献饼图。

5. 奖金决议
   - 等待奖金结果。
   - 未获得奖金。
   - 已获得奖金，并按冻结比例计算建议付款金额。

6. 申诉与审计
   - 成员可对任务贡献提出申诉。
   - 复核人可接受或驳回。
   - 关键操作写入审计日志。

## 贡献公式

```text
任务贡献点 =
加权难度 × 100 × 完成度 × 质量系数 × 准时系数 × 证据系数
```

```text
加权难度 =
主策难度 × 0.5 + 成员难度 × 0.35 + AI 难度 × 0.15
```

```text
最终贡献 =
任务贡献归一化 × 70%
+ 互评贡献归一化 × 20%
+ 关键职责归一化 × 10%
```

多人任务分配：

```text
负责人 70%
协作者均分 30%
```

## 第二阶段建议

- Supabase Auth 和 Row Level Security。
- GitHub App webhook 自动生成证据。
- OpenAI Structured Outputs 任务难度评估。
- pgvector Markdown 文档检索。
- 真实审计日志哈希链。
- 团队协议、签署记录和付款确认。
